🐛 المشكلة 1: الفيديو ما اشتغل (شاشة سوداء)
شايف في الصورة الأولى شاشة سوداء كاملة مع زر تخطي. هذا يعني الفيديو ما تحمّل أو ما بدأ التشغيل.
الأسباب المحتملة:
1. الفيديوهات غير موجودة في المسار الصحيح (/public/videos/)
2. أسماء الملفات مختلفة عن المتوقع (ceramic.mp4, sanitary.mp4, decorative.mp4)
3. خاصية autoPlay ما اشتغلت (بعض المتصفحات تمنعها)
4. الفيديو يحتاج muted + playsInline عشان يشتغل تلقائياً

🐛 المشكلة 2: الكتالوج يظهر قبل الانتقال (الأخطر)
شايف في الصورة الثانية الكتالوج يظهر خلف شاشة Splash، وأنت ما زلت في مرحلة Onboarding. هذي مشكلة كبيرة في الـ flow.
السبب:
Onboarding component لا يحجب الكتالوج بشكل كامل. يعني:

الكتالوج يتحمّل أول
Onboarding يظهر فوقه كـ overlay
بدل إنه يمنع الكتالوج من التحميل حتى ينتهي Onboarding

## المشكلة 1: الفيديوهات لا تشتغل

تشخيص:
- عند الانتقال إلى Stage 2 (CinematicTour)، تظهر شاشة سوداء بدون فيديو
- زر التخطي يظهر، بس الفيديو لا يبدأ

الإصلاحات المطلوبة:

1. **تأكد من مسار الفيديوهات:**
   - تحقق أن الفيديوهات موجودة في /public/videos/
   - الأسماء يجب أن تكون: ceramic.mp4, sanitary.mp4, decorative.mp4
   - إذا الأسماء مختلفة، صحّحها

2. **خصائص عنصر الفيديو (CRITICAL):**
   كل عنصر `<video>` يجب أن يحتوي على:
```jsx
   <video
     autoPlay
     muted
     playsInline
     preload="auto"
     onEnded={handleVideoEnd}
     onError={(e) => console.error('Video error:', e)}
     className="w-full h-full object-cover"
   >
     
   
```
   
   خصائص إلزامية:
   - `autoPlay` — يبدأ التشغيل تلقائياً
   - `muted` — إلزامي لتشغيل autoplay في المتصفحات
   - `playsInline` — يمنع fullscreen على iOS
   - `preload="auto"` — يحمّل الفيديو مسبقاً

3. **إضافة console.log للتشخيص:**
```jsx
   onLoadStart={() => console.log('Video loading started')}
   onCanPlay={() => console.log('Video can play')}
   onPlay={() => console.log('Video is playing')}
   onError={(e) => console.error('Video error:', e.target.error)}
```

4. **إضافة poster للصور الثابتة:**
   - استخرج أول frame من كل فيديو كصورة JPG
   - أو استخدم صور stills موجودة
   - أضف `poster="/images/ceramic-poster.jpg"` لكل فيديو
   - هذا يضمن ظهور صورة فوراً حتى لو الفيديو ما اشتغل

## المشكلة 2: Onboarding لا يحجب الكتالوج

تشخيص:
- عند الضغط على "ادخل الكتالوج" في Stage 1، الكتالوج يظهر خلف الشاشة
- هذا يعني OnboardingFlow يُعرض فوق الكتالوج كـ overlay بدلاً من منع تحميله

الإصلاح المطلوب:

في App.tsx أو المسار `/catalog`:
```jsx
function CatalogRoute() {
  const [showOnboarding, setShowOnboarding] = useState(() => {
    return localStorage.getItem('baytalebaa_onboarding_seen') !== 'true';
  });

  const handleOnboardingComplete = () => {
    localStorage.setItem('baytalebaa_onboarding_seen', 'true');
    setShowOnboarding(false);
  };

  // CRITICAL: استخدم conditional rendering بدلاً من overlay
  if (showOnboarding) {
    return ;
  }

  return ;
}
```

المبدأ:
- إذا showOnboarding === true → ارجع OnboardingFlow فقط (لا تحمّل الكتالوج)
- إذا showOnboarding === false → ارجع ProductCatalogPage فقط
- لا تستخدم `z-index` overlay — استخدم conditional rendering

## اختبار الإصلاحات:

بعد التنفيذ:

1. افتح DevTools → Console
2. امسح localStorage: `localStorage.clear()`
3. أعد تحميل الصفحة
4. يجب أن تظهر Stage 1 فقط (بدون الكتالوج خلفها)
5. اضغط "ادخل الكتالوج" → يجب أن تنتقل لـ Stage 2
6. راقب Console: يجب تظهر رسائل "Video loading started" و "Video can play" و "Video is playing"
7. إذا ظهرت رسالة error، أخبرني بالخطأ بالتحديد
