## CRITICAL PROBLEMS TO FIX

### Problem 1: No "Real Book" Feel
The current design looks like a web page, not a book. Fix:
- Add **paper texture** to every page — use a subtle CSS noise/grain pattern or an off-white `#FAF8F3` background with a very faint repeating texture
- Add a **subtle inner shadow** on page edges to simulate page depth
- The book spine area (center where pages meet) needs a **vertical gradient shadow** to look like a real book binding
- Page corners should feel tangible — slight rounded corners (3-4px only) with a hairline border
- Overall, the book container itself should have a slight `perspective` transform and drop shadow as if it's sitting on a desk

### Problem 2: Cover Page is Too Empty and Plain
Current cover has too much empty dark space with tiny centered content. Fix:
- Make it look like a **real catalog cover** — think IKEA, Saudi Ceramics, or Al Rajhi annual report
- The cover should have a **full-bleed background image** of premium building materials (construction site, marble texture, modern building facade) with a dark gradient overlay
- Company logo should be **large and prominent** — at least 120px wide
- Title "بيت الإباء" in large bold Arabic text (32-40px)
- Subtitle "للمواد البنائية والإنشائية" below
- "كتالوج المنتجات ٢٠٢٥ | Product Catalog 2025" at the bottom
- Add a **gold foil-style** thin border frame (2px solid gold with 24px margin from edges)
- The "PDC" text at the bottom should be removed or replaced with "مركز بيانات المنتجات | Product Data Center" in tiny text
- NO floating animations, NO glowing text — static elegance

### Problem 3: About Us Page is Cramped and Unorganized
Current "من نحن" page has too much text crammed together. Fix:
- Split into **TWO separate pages** (a spread):
  - **Left page**: Company overview text + key stats (20+ سنة خبرة, 5000+ منتج, 500+ عميل, 10+ مستودعات) in a clean grid
  - **Right page**: Vision, Mission, Values + "لماذا بيت الإباء" section
- Use **print magazine typography**: larger body text (13-14px), generous line-height (1.9), proper paragraph spacing
- Section headers should use gold color with a thin underline — not badges or chips
- Stats should be displayed as large numbers with labels below, in a horizontal row with dividers
- Values should be simple text list or minimal pills — NOT colorful buttons
- Add subtle **decorative elements**: thin gold corner brackets, a small ornamental divider between sections
- Content should breathe — generous margins (24-32px padding)

### Problem 4: Product Pages Look Like a Basic Grid, Not a Catalog Page
Current product grid is too plain — just cards in a 2x2 grid with no magazine feel. Fix:
- Add a **page header** on each product page showing the category name + small icon
- Product cards should have:
  - Clean white/light background card with thin (1px) border
  - Product image taking ~60% of card height
  - Name in bold Arabic, SKU in light gray monospace below
  - Price with gold color if available
  - Subtle hover lift effect
- Add a **thin gold line** at the top of the page as a section indicator
- Page footer: centered page number in classic book style — "— ٧ —" with dashes
- Category name should appear as a **running header** (small text at top of page)
- Background of product pages: warm off-white `#FAF8F3` (not pure white, not dark)

### Problem 5: Category Chapter Dividers Need Polish
The current chapter pages are decent but need refinement:
- Keep the background image concept but improve the overlay:
  - Use a more sophisticated gradient: `linear-gradient(160deg, rgba(10,22,40,0.85) 0%, rgba(10,22,40,0.7) 50%, rgba(10,22,40,0.85) 100%)`
  - Add a subtle **vignette** effect (radial gradient darkening at edges)
- Chapter number should be in a **refined gold circle** with thin border, not a filled circle
- Category name: large, bold, with a thin gold line below (not a thick bar)
- Product count: subtle text below, not in a pill/badge
- Remove the small square icon — use the emoji/icon at a reasonable size (48px)
- Add English category name as a smaller subtitle below the Arabic name
- Overall: this should look like a chapter opening in a National Geographic or Architectural Digest

### Problem 6: Table of Contents Needs Classic Magazine Style

- Each entry: "الفصل ١  |  السيراميك والبورسلان .......................... ٥"

### Problem 7: No Bilingual Support
Add full Arabic/English support:

## BRAND IDENTITY

- **Colors**:
  - Primary dark: `#0B1A2E` (deep navy)
  - Surface dark: `#111D32`
  - Gold: `#C8A84B` (primary accent)
  - Gold dark: `#8B7230` (for light mode contrast)
  - Paper/Light bg: `#FAF8F3` (warm off-white, like real paper)
  - Text primary: `#F5F0E8` (dark mode) / `#1A1A2E` (light mode)
  - Text secondary: `#9CA3AF` (dark) / `#6B7280` (light)

- **Typography**:
  - Font: `'29LT Bukra', 'Tajawal', 'Segoe UI', sans-serif`
  - Headings: weight 800, tight letter-spacing
  - Body: weight 400-500, line-height 1.8-1.9
  - Captions: weight 500, size 10-11px
  - Page numbers: classic serif-style or monospace

- **Design language**:
  - Thin gold lines as decorative elements (1-2px)
  - Corner bracket decorations (subtle, in gold at 30% opacity)
  - No gradients on text
  - No glowing/pulsing animations
  - Minimal border-radius (4-8px max on cards, 2-3px on pages)
  - Generous whitespace
  - Print-inspired layouts (columns, running headers, page footers)

## TECHNICAL CONSTRAINTS

- Single file: `src/pages/FlipbookPage.tsx` with default export
- Using: `react-pageflip` (HTMLFlipBook), `@tanstack/react-query`, `react-router-dom`, `lucide-react`
- API: `productsAPI.list()` and `categoriesAPI.list()` from `@/api/client`
- Types: `Product`, `Category` from `@/types`
- Theme: `useThemeStore` from `@/store/themeStore` (zustand) — has `theme` and `toggleTheme`
- All inline styles (project convention — NO CSS modules, NO Tailwind)
- Must remain fully responsive (mobile: portrait single-page, desktop: larger book)
- Keep ALL existing data fetching logic, API calls, and navigation intact
- The flipbook engine handles page flipping — we only style the page contents

## ANTI-PATTERNS (what to avoid)

- ❌ Neon glows, particle effects, cosmic backgrounds
- ❌ Cards with thick colored borders or heavy shadows
- ❌ Large emoji as section icons (use small, subtle)
- ❌ Gradient text or animated text effects
- ❌ Excessive use of backdrop-filter blur
- ❌ Cramming too much content on one page
- ❌ Using bright contrasting colors (stick to navy + gold + warm neutrals)
- ❌ Inconsistent spacing or font sizes
- ❌ Anything that screams "made by AI" or "generic SaaS template"
- ❌ Pure white (#fff) backgrounds — use warm off-white (#FAF8F3) for paper feel
