markdown# Project: Bayt Alebaa Catalog — Cinematic Onboarding Experience

## Overview
Build a cinematic onboarding experience that plays before the main catalog 
loads. The experience consists of three sequential stages: a branded splash 
screen, a three-video cinematic tour, and a welcome transition screen. The 
goal is to create a premium, immersive "wow" moment that positions the 
catalog as an extension of the physical showroom experience.

## Tech Stack
- React (with TypeScript)
- Tailwind CSS for styling
- Framer Motion for animations
- HTML5 `<video>` element for video playback
- localStorage to track first-time visitors

## Project Structure
```
src/
  components/
```

## Stage 1: Splash Screen (3 seconds)

### Visual Design
- **Background:** Linear gradient 
  `linear-gradient(180deg, #0F2530 0%, #1B3D4F 100%)`
- **Floating particles:** 20-30 small circular bubbles floating slowly 
  upward across the screen
  - Color: `#2A5A75` with 40% opacity
  - Size: varies between 4px and 12px
  - Animation: Float upward slowly (20-30 seconds per bubble to cross 
    the screen), with subtle horizontal drift
  - Use Framer Motion for smooth continuous animation
- **Logo:** Centered, use `logo.png` with CSS filter to make it white 
  on dark background: `filter: brightness(0) invert(1);`
  - Appears with fade-in + scale animation (0.9 → 1.0) over 0.8 seconds
  - Size: 120px on desktop, 80px on mobile
- **Tagline below logo:** Arabic text "حيث يبدأ كل تفصيل"
  - Font: IBM Plex Sans Arabic or Tajawal, weight 300 (light)
  - Color: white with 85% opacity
  - Appears 0.4 seconds after logo with fade-in from below 
    (translate-y 10px → 0)
  - Size: text-xl on mobile, text-2xl on desktop
- **CTA Button:** Appears 2 seconds after page load
  - Text: "ادخل المعرض"
  - Style: Transparent background with 2px white border, white text, 
    rounded-full, padding px-8 py-3
  - Hover: background fills with `#2A5A75`, smooth transition
  - Icon: Left-pointing arrow (ArrowLeft from Lucide React) since Arabic 
    is RTL — arrow comes after text in the visual order
  - Animation: fade-in + scale from 0.95 to 1.0

### Interactions
- Clicking "ادخل المعرض" transitions to Stage 2
- If user doesn't click, button stays visible (don't auto-advance)

### Transition Out
- Fade to black over 0.4 seconds before Stage 2 begins

## Stage 2: Cinematic Tour (15 seconds)

### Structure
Three videos play in sequence, each 5 seconds long, with text overlays 
and smooth transitions between them.

### Video Playback Component
- Full-screen video element (`object-fit: cover`)
- Videos auto-play, muted, no controls visible
- Use three `<video>` elements with conditional rendering (one at a time)
- Preload all three videos on Stage 2 mount for smooth transitions

### Video 1 — Ceramic Section (0:00 - 0:05)
- **File:** `/assets/videos/ceramic.mp4`
- **Text overlay:** "الجودة تبدأ من اللمسة الأولى"
  - Position: Bottom-center, 15% from bottom edge
  - Font: IBM Plex Sans Arabic or Tajawal, weight 500
  - Size: text-2xl on mobile, text-3xl on tablet, text-4xl on desktop
  - Color: white
  - Text-shadow: `0 2px 8px rgba(0, 0, 0, 0.6)` for readability
  - Background: subtle dark gradient overlay behind text 
    (linear-gradient from transparent to rgba(0,0,0,0.5))
  - Animation: fade-in from below at 0:00.5 (translate-y 20px → 0, 
    opacity 0 → 1), fade-out at 0:04.2

### Transition 1 (0:05 - 0:05.3)
- Quick white flash: screen briefly flashes to white 
  (opacity 0 → 0.8 → 0) over 0.3 seconds
- Use Framer Motion overlay div with white background and z-index higher 
  than video

### Video 2 — Sanitary/GROHE Section (0:05 - 0:10)
- **File:** `/assets/videos/sanitary.mp4`
- **Text overlay:** "تفاصيل تصنع الفرق"
  - Same styling as Video 1 text
  - Animation: fade-in at 0:05.5, fade-out at 0:09.2

### Transition 2 (0:10 - 0:10.3)
- Same white flash transition

### Video 3 — Decorative Section (0:10 - 0:15)
- **File:** `/assets/videos/decorative.mp4`
- **Text overlay:** "إبداع يتحدث بصمت"
  - Same styling as Video 1 text
  - Animation: fade-in at 0:10.5, fade-out at 0:14.2

### Transition Out (0:15 - 0:15.5)
- Fade to warm beige `#F5F1EA` (opacity 0 → 1) over 0.5 seconds before 
  Stage 3

### Skip Button
- Position: Top-left corner (for LTR-safe skip placement)
- Style: Semi-transparent white background (rgba(255,255,255,0.15)) with 
  backdrop-blur, white text, rounded-full, padding px-4 py-2
- Text: "تخطي" with ChevronLeft icon from Lucide React
- Always visible during Stage 2 with fade-in on Stage 2 mount
- Clicking skips directly to Stage 3

## localStorage Logic
```typescript
const ONBOARDING_KEY = 'baytalebaa_onboarding_seen';

// On app mount, check if user has seen onboarding
const hasSeenOnboarding = localStorage.getItem(ONBOARDING_KEY) === 'true';

// If true → skip onboarding entirely, go to catalog
// If false/null → show onboarding

// When user completes or skips onboarding
localStorage.setItem(ONBOARDING_KEY, 'true');
```

Add a developer-only "Reset Onboarding" button in the catalog footer 
(for testing) that clears localStorage and reloads the page.

## RTL Support

- Set `dir="rtl"` on the root container
- Use Tailwind RTL utilities where needed
- Arrows in buttons should point LEFT (since user moves forward = left 
  in RTL)
- Text alignment: center for onboarding stages (since content is centered)

---

## Responsive Design

- Mobile-first approach
- Videos should maintain full-screen on mobile (object-cover)
- Text sizes scale responsively using Tailwind breakpoints (sm:, md:, lg:)
- Logo scales: 80px mobile, 120px desktop (Stage 1) / 70px mobile, 100px 
  desktop (Stage 3)
- Buttons scale padding on mobile vs desktop

---

## Performance Considerations

- Preload videos when Stage 1 mounts (use `<link rel="preload">` or 
  video preload="auto")
- Add `poster` attribute to video elements with first-frame image for 
  instant visual feedback
- Ensure videos are web-optimized MP4 (H.264 codec, max 2MB each)
- Use `will-change: transform, opacity` on animated elements for 
  smoother animations

---

## Accessibility

- Add `aria-label` to skip button
- Respect `prefers-reduced-motion` media query:
  - Disable particle animations
  - Simplify transitions to basic fades (no scale/translate)
  - Keep videos but remove auto-advance complexity
- Ensure all buttons are keyboard-accessible (Enter/Space to activate)
- Add proper ARIA labels in Arabic

## Expected User Flow

1. User opens the website for the first time
2. Dark teal-navy splash screen appears with floating lighter-blue bubbles 
   and the Bayt Alebaa logo (in white)
3. User reads the tagline "حيث يبدأ كل تفصيل" and clicks "ادخل المعرض"
4. Screen fades to black, then full-screen cinematic tour begins
5. Three videos play sequentially with Arabic text overlays and white 
   flash transitions between them
6. Tour ends with fade to warm beige welcome screen
7. Logo appears in original brand color with welcome message
8. User clicks "ابدأ الجولة" and enters the main catalog
9. localStorage saves that user has seen onboarding
10. On next visit, onboarding is skipped automatically and user goes 
    directly to the catalog
