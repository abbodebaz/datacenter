# إضافة حاسبتين جديدتين لصفحة الخدمات

## السياق
صفحة `/services` موجودة حالياً وفيها:
- كرت "اقتراح منتج" (موجود)
- كرت "حاسبة البلاط والسيراميك" (موجود)

المطلوب: إضافة كرتين جديدين لنفس الصفحة، كل واحد يفتح modal حاسبة 
بوضعين (عادي ومتقدم) مثل حاسبة البلاط.

---

## الكرت الثالث: حاسبة الباركيه والفوم

### بيانات الكرت
- **الأيقونة**: `Layers` من lucide-react
- **العنوان**: "حاسبة الباركيه والفوم"
- **الوصف**: "احسب كمية الباركيه والفوم العازل اللي تحتاجها لأرضية 
  مشروعك"
- **الزر**: "احسب الآن"
- **الإجراء**: يفتح `ParquetCalculatorModal`

### المكون: ParquetCalculatorModal

نفس بنية TileCalculatorModal — Modal مع Tabs (عادية | متقدمة).

---

### الحاسبة العادية — الباركيه (BasicParquetCalculator.tsx)

#### المدخلات:
1. **طريقة البيع** (select):
   - متر مربع
   - باكيت
   - كرتون

2. **إذا اختار "باكيت" أو "كرتون"**:
   - input: "مساحة التغطية للباكيت/الكرتون الواحد (m²)"
   - مثال: 2.4 m²

3. **الطول** (متر)
4. **العرض** (متر)

5. **هل تريد حساب الفوم العازل؟** (checkbox)
   - إذا نعم: المدخلات الإضافية:
     - input: "مساحة رولة الفوم الواحدة (m²)" — default: 15
     - checkbox: "إضافة هدر 5% للفوم"

#### الثوابت:
```typescript
const PARQUET_WASTE_PERCENT = 8;  // ثابت للحاسبة العادية
const FOAM_WASTE_PERCENT = 5;
```

#### المعادلات:
```typescript
// الباركيه
const roomArea = length * width;
const parquetWithWaste = roomArea * (1 + PARQUET_WASTE_PERCENT / 100);

let parquetQuantity = 0;
let parquetUnit = '';

if (saleType === 'square_meter') {
  parquetQuantity = parquetWithWaste;
  parquetUnit = 'm²';
} else if (saleType === 'pack') {
  parquetQuantity = Math.ceil(parquetWithWaste / packCoverage);
  parquetUnit = 'باكيت';
} else if (saleType === 'carton') {
  parquetQuantity = Math.ceil(parquetWithWaste / cartonCoverage);
  parquetUnit = 'كرتون';
}

// الفوم (إذا مفعّل)
let foamRolls = 0;
if (includeFoam) {
  const foamArea = includeFoamWaste 
    ? roomArea * (1 + FOAM_WASTE_PERCENT / 100)
    : roomArea;
  foamRolls = Math.ceil(foamArea / foamRollSize);
}
```

#### النتيجة:
```
📐 مساحة الغرفة: X m²
📦 مع الهدر (8%): X m²

🎯 الباركيه: X [باكيت/كرتون/m²]

[إذا الفوم مفعّل:]
🧴 الفوم العازل: X رولة
```

### الحاسبة المتقدمة — الباركيه (AdvancedParquetCalculator.tsx)

#### المدخلات:
1. **طريقة البيع** (select):
   - متر مربع
   - باكيت
   - كرتون

2. **إذا باكيت/كرتون**: مساحة التغطية للوحدة

3. **أبعاد الغرفة**:
   - الطول (متر)
   - العرض (متر)

4. **شكل الغرفة** (select):
   - مستطيلة (default)
   - حرف L → يظهر inputs إضافية:
     - طول القسم الثاني
     - عرض القسم الثاني

5. **عدد الأعمدة/الفتحات الداخلية** (optional, default: 0)
   - مساحة متوسطة للعمود الواحد: 0.25 m²

6. **اتجاه التركيب** (select):
   - موازي للجدار (هدر عادي)
   - قطري (هدر أكبر)
   - متعرج/Herringbone (هدر كبير)

7. **نسبة الهدر** (select):
   - 8% — تركيب عادي
   - 12% — تركيب قطري
   - 15% — Herringbone
   - مخصص

8. **الفوم العازل** (section قابل للطي):
   - checkbox: "إضافة حساب الفوم"
   - إذا مفعّل:
     - نوع الفوم (select): عادي 2mm | متوسط 3mm | سميك 5mm
     - مساحة رولة الفوم (default: 15 m²)
     - نسبة هدر الفوم (default: 5%)

#### المعادلات:
```typescript
// حساب المساحة الأساسية
let totalArea = length * width;

// إضافة مساحة إضافية لشكل L
if (roomShape === 'L') {
  totalArea += secondLength * secondWidth;
}

// خصم الأعمدة
const columnsArea = columns * 0.25;
const netArea = Math.max(0, totalArea - columnsArea);

// حساب الباركيه مع الهدر
const parquetWithWaste = netArea * (1 + wastePercent / 100);

let parquetQuantity = 0;
let parquetUnit = '';

if (saleType === 'square_meter') {
  parquetQuantity = parquetWithWaste;
  parquetUnit = 'm²';
} else if (saleType === 'pack') {
  parquetQuantity = Math.ceil(parquetWithWaste / packCoverage);
  parquetUnit = 'باكيت';
} else if (saleType === 'carton') {
  parquetQuantity = Math.ceil(parquetWithWaste / cartonCoverage);
  parquetUnit = 'كرتون';
}

// الفوم
let foamRolls = 0;
let foamArea = 0;
if (includeFoam) {
  foamArea = netArea * (1 + foamWastePercent / 100);
  foamRolls = Math.ceil(foamArea / foamRollSize);
}
```

#### النتيجة (مفصّلة):
```
📐 تفاصيل الغرفة:
  • الأبعاد: X × Y m
  • شكل الغرفة: [مستطيلة/حرف L]
  • المساحة الإجمالية: X m²
  • خصم الأعمدة: -X m²
  • المساحة الصافية: X m²

📦 الباركيه:
  • نسبة الهدر: X%
  • الكمية مع الهدر: X m²
  • عدد [الباكيت/الكراتين]: X
  • اتجاه التركيب: [...]

[إذا الفوم مفعّل:]
🧴 الفوم العازل:
  • النوع: [...]
  • المساحة مع الهدر: X m²
  • عدد الرولات: X
```

## الكرت الرابع: حاسبة الغراء

### بيانات الكرت
- **الأيقونة**: `Droplet` أو `Beaker` من lucide-react
- **العنوان**: "حاسبة الغراء"
- **الوصف**: "احسب كمية الغراء اللي تحتاجها لتركيب البلاط أو الباركيه"
- **الزر**: "احسب الآن"
- **الإجراء**: يفتح `GlueCalculatorModal`

### المكون: GlueCalculatorModal

نفس بنية الحاسبات الثانية — Modal مع Tabs (عادية | متقدمة).

---

### الحاسبة العادية — الغراء (BasicGlueCalculator.tsx)

#### المدخلات:
1. **المساحة** (m²) — input مباشر
2. **معدل الاستهلاك** (kg/m²) — المستخدم يدخله بنفسه
   - placeholder: "مثلاً: 4"
   - hint: "عادة بين 3-6 كجم لكل متر مربع"
3. **وزن الكيس الواحد** (kg) — default: 25

#### الثوابت:
```typescript
const GLUE_WASTE_PERCENT = 10;
```

#### المعادلات:
```typescript
const totalGlueKg = area * consumptionRate * 1.10;  // مع هدر 10%
const bagsNeeded = Math.ceil(totalGlueKg / bagWeight);
const actualKg = bagsNeeded * bagWeight;
```

#### النتيجة:
```
📐 المساحة: X m²
⚖️ الاستهلاك الكلي: X kg
📦 عدد الأكياس: X كيس
💼 الوزن الفعلي: X kg
```

---

### الحاسبة المتقدمة — الغراء (AdvancedGlueCalculator.tsx)

#### المدخلات:
1. **نوع السطح المراد تركيبه** (select):
   - سيراميك
   - بورسلان
   - رخام
   - باركيه
   - فسيفساء/موزاييك
   - حجر طبيعي

2. **مقاس البلاطة/القطعة** (select):
   - صغير (أقل من 30×30)
   - متوسط (30×30 إلى 60×60)
   - كبير (60×60 إلى 80×80)
   - كبير جداً (أكبر من 80×80)

3. **مكان التركيب** (select):
   - داخلي
   - خارجي
   - رطب (حمامات/مطابخ)
   - مسابح

4. **المساحة** (m²)

5. **معدل الاستهلاك** (kg/m²):
   - يظهر اقتراح تلقائي بناءً على الخيارات السابقة (الجدول أدناه)
   - المستخدم يقدر يعدّله

6. **وزن الكيس** (kg) — default: 25

7. **نسبة الهدر** (select):
   - 10% (عادي)
   - 15% (تركيب صعب)
   - 20% (أسطح غير منتظمة)
   - مخصص

#### جدول الاستهلاك المقترح:
```typescript
const GLUE_CONSUMPTION_TABLE = {
  // [نوع السطح][مقاس القطعة] = kg/m²
  ceramic: {
    small: 3.0,
    medium: 4.0,
    large: 5.0,
    xlarge: 6.0,
  },
  porcelain: {
    small: 3.5,
    medium: 4.5,
    large: 5.5,
    xlarge: 6.5,
  },
  marble: {
    small: 4.0,
    medium: 5.0,
    large: 6.0,
    xlarge: 7.0,
  },
  parquet: {
    small: 2.5,
    medium: 3.0,
    large: 3.5,
    xlarge: 4.0,
  },
  mosaic: {
    small: 3.5,
    medium: 4.0,
    large: 4.5,
    xlarge: 5.0,
  },
  natural_stone: {
    small: 5.0,
    medium: 6.0,
    large: 7.0,
    xlarge: 8.0,
  },
};

// تعديل حسب المكان
const LOCATION_MULTIPLIER = {
  indoor: 1.0,
  outdoor: 1.15,
  wet: 1.2,
  pool: 1.3,
};
```

#### المعادلات:
```typescript
// الاستهلاك المقترح
const baseRate = GLUE_CONSUMPTION_TABLE[surfaceType][tileSize];
const suggestedRate = baseRate * LOCATION_MULTIPLIER[location];

// الاستهلاك الفعلي (المستخدم يقدر يعدّله)
const actualRate = userRate || suggestedRate;

// الكمية الكلية
const totalGlueKg = area * actualRate * (1 + wastePercent / 100);
const bagsNeeded = Math.ceil(totalGlueKg / bagWeight);
const actualKg = bagsNeeded * bagWeight;
const surplusKg = actualKg - totalGlueKg;
```

#### النتيجة (مفصّلة):
```
📋 تفاصيل المشروع:
  • نوع السطح: [...]
  • مقاس القطعة: [...]
  • مكان التركيب: [...]
  • المساحة: X m²

⚖️ حسابات الغراء:
  • معدل الاستهلاك المقترح: X kg/m²
  • المعدل المستخدم: X kg/m²
  • نسبة الهدر: X%
  • الاستهلاك الكلي: X kg

📦 الكميات:
  • عدد الأكياس: X كيس (X kg لكل كيس)
  • الوزن الفعلي: X kg
  • الفائض: X kg

💡 ملاحظة: الاستهلاك يختلف حسب نوع البلاط ومهارة الفني 
  ودرجة استواء السطح
```

## ملاحظات مهمة

- **نفس التصميم** لكل الحاسبات (Modal + Tabs + live calculation + 
  Framer Motion animations)
- **نفس الألوان** (#1B3D4F primary, #2A5A75 secondary, #FAF7F2 
  background)
- **نفس الخطوط** (IBM Plex Sans Arabic / Tajawal)
- **حساب فوري** — بدون زر "احسب"
- **Validation موحد** — لا قيم سالبة، حدود معقولة، عرض "—" للمدخلات 
  الفارغة
- **بدون أسعار** — فقط الكميات
- **بدون API** — كل الحسابات في الـ frontend
- **مرن للتعديل** — الثوابت (نسب الهدر، جداول الاستهلاك) في أعلى كل 
  ملف عشان تعدّلها بسهولة لاحقاً
- **Responsive** — Modal full-screen على الموبايل، max-width 700px على 
  الديسكتوب
